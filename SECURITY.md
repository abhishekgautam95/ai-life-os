# Security Policy

## Supported Scope

AI Life OS is currently an early-stage open-source MVP.

Security fixes are most likely to be accepted for:

- authentication
- access control
- sensitive data exposure
- dependency vulnerabilities with clear impact

## Reporting a Vulnerability

Please do not open a public GitHub issue for security vulnerabilities.

Instead:

1. Share the issue privately with the maintainer if a private contact path exists
2. Include clear reproduction steps
3. Include impact and affected area
4. Include possible mitigation if known

## What to Include

Please include:

- affected route or file
- steps to reproduce
- expected behavior
- actual behavior
- severity estimate
- screenshots or logs if helpful

## Response Expectations

Because this is an independent MVP project, response times may vary.

Good reports will still be reviewed seriously and fixed as time allows.
